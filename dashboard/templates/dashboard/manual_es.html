{# 1. Presentación #}

1. Presentación de la plataforma

KoboDashBoard es una aplicación web que se conecta a un servidor KoboToolBox y muestra los datos de envío de formularios en forma de paneles de control interactivos.

La plataforma se organiza en grupos: cada grupo dispone de un conjunto de formularios accesibles y una lista de miembros. Un usuario solo ve los formularios pertenecientes a sus grupos.

RolAcceso
Usuario Consulta los dashboards de sus grupos, usa filtros y exportaciones.
Admin de grupo Todo lo que puede hacer un usuario + gestionar los miembros de su grupo + cargar módulos de dashboard.
Administrador principal Acceso completo: configuración del servidor, gestión de todos los formularios, grupos y usuarios.
{# 2. Acceso #}

2. Acceso e inicio de sesión

Solicitar acceso

Si aún no tiene cuenta, haga clic en «Crear una cuenta» en la página de inicio de sesión. Introduzca su nombre completo y dirección de correo electrónico, luego elija una contraseña. Su solicitud se enviará a un administrador para su validación. Recibirá un mensaje de confirmación una vez activada su cuenta.

Iniciar sesión

Introduzca su dirección de correo electrónico y contraseña en la página de inicio de sesión (/accounts/login/). La sesión permanece abierta hasta que haga clic en «Cerrar sesión» en la parte superior derecha.

Contraseña olvidada

Contacte a su administrador de grupo o al administrador principal. Este puede generar un enlace de restablecimiento de contraseña y enviárselo directamente.

{# 3. Dashboards #}

3. Consultar los dashboards

La página de inicio («Formularios») lista todos los dashboards a los que tiene acceso. Haga clic en «Ver dashboard» para abrir un formulario.

Existen tres tipos de vistas según el formulario:

  • Dashboard configurado — construido mediante el editor de dashboard (ver sección 8). Muestra los widgets definidos por el administrador: cifras resumen, gráficos de sectores o de columnas, tablas de datos, con filtros personalizados.
  • Dashboard de indicadores — disponible si se ha cargado un módulo Python específico para el formulario. Muestra tarjetas resumen, gráficos por país/período/resultado, desagregación y una tabla de datos detallada.
  • Matriz de cobertura — disponible si se ha cargado un módulo Python específico. Muestra una tabla cruzada actividades × países con el número de envíos por combinación. Las celdas son clicables y dan acceso a la lista de envíos correspondientes.

El botón Actualizar (en la página del dashboard) fuerza la recarga de datos desde el servidor KoboToolBox, ignorando la caché.

{# 4. Filtros #}

4. Usar los filtros

Los dashboards ofrecen una barra de filtros en la parte superior de la página. Los filtros disponibles varían según el tipo de formulario:

  • País — restringe la visualización a un país o área geográfica.
  • Año / Trimestre — filtra por período de informe.
  • Resultado — muestra solo los indicadores de un resultado dado.
  • Responsable — filtra por persona o entidad responsable de la actividad (matriz de cobertura).

Los filtros son acumulables. Para restablecerlos, haga clic en «Restablecer» o elimine los parámetros de la URL.

{# 5. Exportaciones #}

5. Exportar datos

Desde la página del dashboard están disponibles dos formatos de exportación:

  • CSV — archivo de texto separado por comas, compatible con la mayoría de hojas de cálculo. Codificación UTF-8 con BOM para compatibilidad óptima con Excel.
  • XLSX — archivo Excel nativo con encabezados formateados y columnas ajustadas automáticamente.

La exportación abarca todos los datos del formulario, no solo las filas filtradas en pantalla.

{# 6. Admin de grupo #}

6. Administradores de grupo

Si es administrador de un grupo, el icono aparece en la barra de navegación. Da acceso a la página «Mi grupo».

Gestionar miembros
  • Añadir un miembro — seleccione un usuario activo en la lista desplegable y haga clic en «Añadir». Solo aparecen los usuarios ya creados y activados por el administrador principal.
  • Desactivar una cuenta — el usuario ya no puede iniciar sesión, pero su cuenta se conserva. Haga clic en .
  • Eliminar una cuenta — eliminación definitiva. Haga clic en y confirme.
  • Restablecer contraseña — haga clic en para generar un enlace de restablecimiento de un solo uso (válido 3 días). Copie el enlace y envíeselo al usuario por correo electrónico o mensajería.
Editor de dashboard

Como administrador de grupo, puede crear y modificar dashboards visuales para los formularios de su grupo mediante el editor de dashboard. Acceda desde la página Configuración o desde la lista de formularios. Ver sección 8 para más detalles.

Gestionar módulos de dashboard

Cada formulario puede tener un archivo de módulo Python (.py) que define cómo se analizan y muestran los envíos. Como admin de grupo, puede:

  • Descargar el módulo actual — haga clic en Módulo para recuperar el archivo .py en uso.
  • Cargar un nuevo módulo — seleccione un archivo .py y haga clic en Cargar. El nuevo módulo tiene efecto inmediato (sin reinicio del servidor en modo desarrollo; se necesita reiniciar gunicorn en producción).
El archivo módulo debe ser un script Python válido con un identificador Python válido como nombre de archivo (letras, dígitos, guiones bajos — sin espacios ni guiones).
{# 7. Administrador principal #}

7. Administrador principal

El administrador principal accede a la página Configuración ( en la barra de navegación).

Conexión al servidor KoboToolBox

Introduzca la URL del servidor (ej. https://kobo.ifrc.org) y el token API. El token está disponible en su perfil KoboToolBox en <url-servidor>/token/?format=json. Haga clic en «Probar y cargar formularios» para verificar la conexión y obtener la lista de formularios disponibles.

Añadir un formulario

Tras cargar la lista de formularios, seleccione uno en el menú desplegable y haga clic en «Añadir este formulario». Puede ajustar la duración de la caché (en segundos) para controlar la frecuencia de sincronización con KoboToolBox.

Gestionar grupos

Cree grupos con nombre para organizar el acceso a los formularios. Para cada grupo, puede:

  • Asignar formularios (los miembros del grupo solo verán esos formularios).
  • Asignar miembros (usuarios activos).
  • Designar administradores de grupo entre los miembros.

Los formularios no asignados a ningún grupo son visibles solo por el administrador principal.

Gestionar usuarios

La página «Gestión de usuarios» () lista las cuentas pendientes de validación y las cuentas activas. Puede:

  • Activar una nueva cuenta tras verificar la identidad.
  • Desactivar temporalmente una cuenta sin eliminarla.
  • Eliminar definitivamente una cuenta.
  • Generar un enlace de restablecimiento de contraseña () — envíeselo al usuario por el canal de su elección. El enlace es válido 3 días y de un solo uso.
Identidad y color

Desde Configuración, en la tarjeta «Identidad», puede:

  • Nombre de la organización — se muestra en el centro de la barra de navegación en lugar de «KoboDashBoard».
  • Logotipo — reemplaza el texto en el centro de la barra de navegación. Formatos recomendados: PNG o SVG transparente, altura 48 px.
  • Color principal — use el selector de color o introduzca un código hexadecimal (ej. #00833D). El botón «Predeterminado» restaura el rojo Bootstrap. El cambio es instantáneo para todos los usuarios.
Idioma de la interfaz

Un menú desplegable en la barra de navegación permite cambiar el idioma de la interfaz. Hay cinco idiomas disponibles: Français, English, Español, العربية (con diseño de derecha a izquierda automático), Русский. La elección se recuerda para la sesión actual. El mismo selector está disponible en las páginas de inicio de sesión e inscripción.

Recuperación de acceso

En caso de pérdida de contraseña, conéctese al servidor por SSH y ejecute: 

python3 manage.py init_admin

Un enlace de inicio de sesión de un solo uso (válido 3 días) se muestra en el terminal.

{# 8. Editor de dashboard #}

8. Editor de dashboard

El editor de dashboard permite a los administradores crear paneles de control visuales para cualquier formulario sin escribir código. Pueden coexistir varios dashboards para un mismo formulario.

El editor es accesible desde la página Configuración, haciendo clic en «Editor de dashboard» junto al formulario correspondiente. Está disponible para el administrador principal y los administradores de grupo cuyo grupo incluye ese formulario.

Lista de dashboards

La lista muestra todos los dashboards creados para el formulario. Puede crear uno nuevo dándole un nombre. Para cada dashboard existente, hay dos acciones disponibles: Ver (visualización de solo lectura) y Editar (apertura en el editor).

Estructura de un dashboard

Un dashboard está compuesto de filas apiladas verticalmente. Cada fila puede contener 1, 2 o 3 columnas. Las filas se pueden reordenar (flechas ▲▼) o eliminar.

En la parte superior del editor, puede:

  • Renombrar el dashboard.
  • Configurar filtros — seleccione los campos del formulario que aparecerán como filtros en la parte superior del dashboard (ej. país, período). Los filtros se pueden reordenar y eliminar. Se puede introducir una etiqueta personalizada para cada uno.
Añadir y editar widgets

Cada fila puede contener widgets. Haga clic en «Añadir un widget» en una fila para abrir el formulario de adición. Hay cuatro tipos de widgets disponibles:

TipoDescripciónOpciones principales
Cifra resumen Muestra un valor único en caracteres grandes. Número de envíos, suma o promedio de un campo numérico.
Gráfico de sectores Gráfico de tarta agrupado por los valores de un campo. Color y etiqueta personalizables por valor.
Gráfico de columnas / líneas Barras o líneas con uno o varios campos de serie. Eje X (opcional), estilo (barras / líneas), modo apilado, series ordenadas con color y etiqueta.
Tabla de datos Muestra los envíos en forma de tabla. Selección de las columnas a mostrar.

Para editar un widget existente, haga clic en el icono en su tarjeta. El formulario de edición se muestra en línea. Para eliminar un widget, haga clic en y confirme.

Configurar las series (gráfico de columnas / líneas)

El widget «Gráfico de columnas / líneas» permite mostrar varios campos como series en un mismo gráfico. En la sección Series:

  • Seleccione un campo en la lista desplegable y haga clic en «+ Añadir».
  • Cada serie aparece en la lista con un selector de color y un campo etiqueta (mostrado en la leyenda).
  • Reordene las series con las flechas ▲▼.
  • Elimine una serie con el botón ×.

Para los campos de elección fija (select_one), una sección adicional «Colores por valor» permite asignar un color específico a cada categoría (ej. Hombre / Mujer / Otro).

Los cambios en el editor se guardan inmediatamente con cada acción (añadir, eliminar, reordenar). El botón Vista previa en la parte superior del editor permite visualizar el dashboard tal como aparecerá a los usuarios.
{# 9. Módulos Python #}

9. Dashboards avanzados con módulos Python

El editor de dashboard cubre los casos habituales. Para análisis más complejos — lógica de negocio específica, agregaciones personalizadas, matrices de cobertura — es posible crear un módulo Python con la ayuda de Claude Code.

Cómo funciona un módulo

Un módulo es un archivo .py ubicado en el directorio form_modules/ del servidor. Está vinculado a un formulario a través de su UID de KoboToolBox y define cómo se analizan y muestran los envíos. Son posibles dos estilos:

  • Dashboard de indicadores — el módulo implementa parse_submissions(). Muestra tarjetas resumen y gráficos por país, período y resultado.
  • Matriz de cobertura — el módulo implementa parse_structure(). Muestra una tabla cruzada actividades × países con el número de envíos por combinación.
Crear un módulo con Claude Code
  1. Abra Claude Code en el directorio del proyecto KoboDashBoard.
  2. Describa el formulario, su UID de KoboToolBox y la lógica deseada.
  3. Claude Code crea form_modules/<nombre>.py basado en la clase FormModule.
  4. Reinicie el servidor de desarrollo — el módulo se registra automáticamente.
  5. Pruebe el dashboard localmente antes de desplegarlo.
Exportar e importar un módulo

Los archivos de módulo no están incluidos en el repositorio Git (para no exponer datos específicos de la organización). Para desplegar un módulo en producción:

  1. Descargar desde el servidor de desarrollo — en Configuración, haga clic en Descargar módulo junto al formulario.
  2. Importar en producción — en el servidor de producción, en Configuración, haga clic en Cargar módulo y seleccione el archivo descargado.
  3. Reinicie gunicorn: sudo systemctl restart kobodashboard (o use el script de despliegue).
Varios dashboards para el mismo formulario

Un formulario puede tener tanto dashboards creados mediante el editor como un módulo Python. Cuando hay varias opciones disponibles, se muestra una página de selección al abrir el formulario, listando todos los dashboards por nombre.

Los módulos Python nunca se publican en GitHub — permanecen en el servidor y se transfieren solo mediante el botón de descarga/carga en Configuración.
