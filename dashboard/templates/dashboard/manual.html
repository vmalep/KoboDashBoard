{# 1. Présentation #}

1. Présentation de la plateforme

KoboDashBoard est une application web qui se connecte à un serveur KoboToolBox et affiche les données de soumission de formulaires sous forme de tableaux de bord interactifs.

La plateforme est organisée en groupes : chaque groupe dispose d'un ensemble de formulaires accessibles et d'une liste de membres. Un utilisateur ne voit que les formulaires appartenant à ses groupes.

RôleAccès
Utilisateur Consulte les dashboards de ses groupes, utilise les filtres et exports.
Admin de groupe Tout ce qu'un utilisateur peut faire + gérer les membres de son groupe + charger des modules de dashboard.
Administrateur principal Accès complet : configuration du serveur, gestion de tous les formulaires, groupes et utilisateurs.
{# 2. Accès et connexion #}

2. Accès et connexion

Demander l'accès

Si vous n'avez pas encore de compte, cliquez sur « Créer un compte » sur la page de connexion. Renseignez votre nom complet et votre adresse email, puis choisissez un mot de passe. Votre demande sera transmise à un administrateur pour validation. Vous recevrez un message de confirmation une fois votre compte activé.

Se connecter

Saisissez votre adresse email et votre mot de passe sur la page de connexion (/accounts/login/). La session reste ouverte jusqu'à ce que vous cliquiez sur « Déconnexion » en haut à droite.

Mot de passe oublié

Contactez votre administrateur de groupe ou l'administrateur principal. Celui-ci peut générer un lien de réinitialisation de mot de passe et vous le transmettre directement.

{# 3. Dashboards #}

3. Consulter les dashboards

La page d'accueil (« Formulaires ») liste tous les dashboards auxquels vous avez accès. Cliquez sur « Visionner le dashboard » pour ouvrir un formulaire.

Trois types de vues existent selon le formulaire :

  • Dashboard configuré — construit via l'éditeur de dashboard (voir section 8). Affiche les widgets définis par l'administrateur : chiffres résumés, graphiques en secteurs ou en colonnes, tableaux de données, avec filtres personnalisés.
  • Dashboard indicateurs — disponible si un module Python spécifique a été chargé pour le formulaire. Affiche des cartes de synthèse, des graphiques par pays/période/résultat, une désagrégation et un tableau de données détaillé.
  • Matrice de couverture — disponible si un module Python spécifique a été chargé. Affiche un tableau croisé activités × pays avec le nombre de soumissions par combinaison. Les cellules cliquables donnent accès à la liste des soumissions correspondantes.

Le bouton Rafraîchir (sur la page du dashboard) force le rechargement des données depuis le serveur KoboToolBox, en ignorant le cache.

{# 4. Filtres #}

4. Utiliser les filtres

Les dashboards proposent une barre de filtres en haut de page. Les filtres disponibles varient selon le type de formulaire :

  • Pays — restreint l'affichage à un pays ou une zone géographique.
  • Année / Trimestre — filtre par période de rapport.
  • Résultat — affiche uniquement les indicateurs d'un résultat donné.
  • Responsable — filtre par personne ou entité responsable de l'activité (matrice de couverture).

Les filtres sont cumulables. Pour les réinitialiser, cliquez sur « Réinitialiser » ou supprimez les paramètres de l'URL.

{# 5. Exports #}

5. Exporter les données

Deux formats d'export sont disponibles depuis la page du dashboard :

  • CSV — fichier texte séparé par des virgules, compatible avec la plupart des tableurs. Encodage UTF-8 avec BOM pour une compatibilité optimale avec Excel.
  • XLSX — fichier Excel natif avec en-têtes formatées et colonnes ajustées automatiquement.

L'export porte sur l'ensemble des données du formulaire, pas uniquement sur les lignes filtrées à l'écran.

{# 6. Admin de groupe #}

6. Administrateurs de groupe

Si vous êtes administrateur d'un groupe, l'icône apparaît dans la barre de navigation. Elle donne accès à la page « Mon groupe ».

Gérer les membres
  • Ajouter un membre — sélectionnez un utilisateur actif dans la liste déroulante et cliquez sur « Ajouter ». Seuls les utilisateurs déjà créés et activés par l'administrateur principal apparaissent dans la liste.
  • Désactiver un compte — l'utilisateur ne peut plus se connecter mais son compte est conservé. Cliquez sur .
  • Supprimer un compte — suppression définitive. Cliquez sur et confirmez.
  • Réinitialiser le mot de passe — cliquez sur pour générer un lien de réinitialisation à usage unique (valable 3 jours). Copiez ce lien et transmettez-le à l'utilisateur par email ou messagerie.
Éditeur de dashboard

En tant qu'administrateur de groupe, vous pouvez créer et modifier des dashboards visuels pour les formulaires de votre groupe via l'éditeur de dashboard. Accédez-y depuis la page Paramètres ou depuis la liste des formulaires. Voir la section 8 pour le détail des fonctionnalités.

Gérer les modules de dashboard

Chaque formulaire peut avoir un fichier de module Python (.py) qui définit comment les soumissions sont analysées et affichées. En tant qu'admin de groupe, vous pouvez :

  • Télécharger le module actuel — cliquez sur Module pour récupérer le fichier .py en cours d'utilisation.
  • Charger un nouveau module — sélectionnez un fichier .py et cliquez sur Charger. Le nouveau module prend effet immédiatement (sans redémarrage du serveur en mode développement ; un redémarrage gunicorn est nécessaire en production).
Le fichier module doit être un script Python valide avec un identifiant Python valide comme nom de fichier (lettres, chiffres, underscores — sans espaces ni tirets).
{# 7. Administrateur principal #}

7. Administrateur principal

L'administrateur principal accède à la page Paramètres ( dans la barre de navigation).

Connexion au serveur KoboToolBox

Renseignez l'URL du serveur (ex. https://kobo.ifrc.org) et le token API. Le token est disponible sur votre profil KoboToolBox à l'adresse <url-serveur>/token/?format=json. Cliquez sur « Tester & charger les formulaires » pour vérifier la connexion et obtenir la liste des formulaires disponibles.

Ajouter un formulaire

Après avoir chargé la liste des formulaires, sélectionnez-en un dans le menu déroulant et cliquez sur « Ajouter ce formulaire ». Vous pouvez ajuster la durée du cache (en secondes) pour contrôler la fréquence de synchronisation avec KoboToolBox.

Gérer les groupes

Créez des groupes nommés pour organiser l'accès aux formulaires. Pour chaque groupe, vous pouvez :

  • Affecter des formulaires (les membres du groupe verront uniquement ces formulaires).
  • Affecter des membres (utilisateurs actifs).
  • Désigner des administrateurs de groupe parmi les membres.

Les formulaires non affectés à aucun groupe sont visibles uniquement par l'administrateur principal.

Gérer les utilisateurs

La page « Gestion des utilisateurs » () liste les comptes en attente de validation et les comptes actifs. Vous pouvez :

  • Activer un nouveau compte après vérification de l'identité.
  • Désactiver temporairement un compte sans le supprimer.
  • Supprimer définitivement un compte.
  • Générer un lien de réinitialisation de mot de passe () — à transmettre à l'utilisateur par le canal de votre choix. Le lien est valable 3 jours et à usage unique.
Identité et couleur

Depuis les Paramètres, dans la carte « Identité », vous pouvez :

  • Nom de l'organisation — s'affiche au centre de la barre de navigation à la place de « KoboDashBoard ».
  • Logo — remplace le texte au centre de la barre de navigation. Formats recommandés : PNG ou SVG transparent, hauteur 48 px.
  • Couleur principale — utilisez le sélecteur ou saisissez un code hexadécimal (ex. #00833D). Le bouton « Défaut » remet le rouge Bootstrap. La modification est instantanée pour tous les utilisateurs.
Langue de l'interface

Un menu déroulant dans la barre de navigation permet de changer la langue de l'interface. Cinq langues sont disponibles : Français, English, Español, العربية (avec mise en page droite-gauche automatique), Русский. Le choix est mémorisé pour la session en cours. Ce même sélecteur est disponible sur les pages de connexion et d'inscription.

Récupération d'accès

En cas de perte de mot de passe, connectez-vous au serveur via SSH et exécutez : 

python3 manage.py init_admin

Un lien de connexion à usage unique (valable 3 jours) s'affiche dans le terminal.

{# 8. Éditeur de dashboard #}

8. Éditeur de dashboard

L'éditeur de dashboard permet aux administrateurs de créer des tableaux de bord visuels pour n'importe quel formulaire, sans écrire de code. Plusieurs dashboards peuvent coexister pour un même formulaire.

L'éditeur est accessible depuis la page Paramètres, en cliquant sur « Éditeur de dashboard » à côté du formulaire concerné. Il est disponible pour l'administrateur principal et les administrateurs de groupe dont le groupe inclut ce formulaire.

Liste des dashboards

La liste affiche tous les dashboards créés pour le formulaire. Vous pouvez en créer un nouveau en lui donnant un nom. Pour chaque dashboard existant, deux actions sont disponibles : Voir (affichage en lecture seule) et Modifier (ouverture dans l'éditeur).

Structure d'un dashboard

Un dashboard est composé de lignes empilées verticalement. Chaque ligne peut contenir 1, 2 ou 3 colonnes. Les lignes peuvent être réordonnées (flèches ▲▼) ou supprimées.

En haut de l'éditeur, vous pouvez :

  • Renommer le dashboard.
  • Configurer les filtres — sélectionnez les champs du formulaire qui apparaîtront comme filtres en haut du dashboard (ex. pays, période). Les filtres peuvent être réordonnés et supprimés. Un libellé personnalisé peut être saisi pour chacun.
Ajouter et modifier des widgets

Chaque ligne peut contenir des widgets. Cliquez sur « Ajouter un widget » dans une ligne pour ouvrir le formulaire d'ajout. Quatre types de widgets sont disponibles :

TypeDescriptionOptions principales
Chiffre résumé Affiche une valeur unique en gros caractères. Nombre de soumissions, somme ou moyenne d'un champ numérique.
Graphique en secteurs Camembert groupé par les valeurs d'un champ. Couleur et libellé personnalisables par valeur.
Graphique en colonnes / lignes Barres ou lignes avec un ou plusieurs champs de série. Axe X (optionnel), style (barres / lignes), mode empilé, séries ordonnées avec couleur et libellé.
Tableau de données Affiche les soumissions sous forme de tableau. Sélection des colonnes à afficher.

Pour modifier un widget existant, cliquez sur l'icône dans sa carte. Le formulaire d'édition s'affiche en ligne. Pour supprimer un widget, cliquez sur et confirmez.

Configurer les séries (graphique en colonnes / lignes)

Le widget « Graphique en colonnes / lignes » permet d'afficher plusieurs champs comme séries sur un même graphique. Dans la section Séries :

  • Sélectionnez un champ dans la liste déroulante et cliquez sur « + Ajouter ».
  • Chaque série apparaît dans la liste avec un sélecteur de couleur et un champ libellé (affiché dans la légende).
  • Réordonnez les séries avec les flèches ▲▼.
  • Supprimez une série avec le bouton ×.

Pour les champs à choix fixe (select_one), une section supplémentaire « Couleurs par valeur » permet d'assigner une couleur spécifique à chaque catégorie (ex. Homme / Femme / Autre).

Les modifications dans l'éditeur sont enregistrées immédiatement à chaque action (ajout, suppression, réordonnancement). Le bouton Aperçu en haut de l'éditeur permet de visualiser le dashboard tel qu'il apparaîtra aux utilisateurs.
{# 9. Modules Python #}

9. Dashboards avancés avec modules Python

L'éditeur de dashboard couvre les cas courants. Pour des analyses plus complexes — logique métier spécifique, agrégations personnalisées, matrices de couverture — il est possible de créer un module Python avec l'aide de Claude Code.

Comment fonctionne un module

Un module est un fichier .py placé dans le répertoire form_modules/ du serveur. Il est lié à un formulaire via son UID KoboToolBox et définit comment les soumissions sont analysées et affichées. Deux styles sont possibles :

  • Dashboard indicateurs — le module implémente parse_submissions(). Affiche des cartes de synthèse et des graphiques par pays, période et résultat.
  • Matrice de couverture — le module implémente parse_structure(). Affiche un tableau croisé activités × pays avec le nombre de soumissions par combinaison.
Créer un module avec Claude Code
  1. Ouvrez Claude Code dans le répertoire du projet KoboDashBoard.
  2. Décrivez le formulaire, son UID KoboToolBox et la logique souhaitée.
  3. Claude Code crée form_modules/<nom>.py basé sur la classe FormModule.
  4. Redémarrez le serveur de développement — le module s'enregistre automatiquement.
  5. Testez le dashboard localement avant de le déployer.
Exporter et importer un module

Les fichiers de module ne sont pas inclus dans le dépôt Git (pour ne pas exposer de données propres à l'organisation). Pour déployer un module en production :

  1. Télécharger depuis le serveur de développement — dans Paramètres, cliquez sur Télécharger le module à côté du formulaire.
  2. Importer en production — sur le serveur de production, dans Paramètres, cliquez sur Charger un module et sélectionnez le fichier.
  3. Redémarrez gunicorn : sudo systemctl restart kobodashboard (ou utilisez le script de déploiement).
Plusieurs dashboards pour un même formulaire

Un formulaire peut avoir à la fois des dashboards créés via l'éditeur et un module Python. Lorsque plusieurs options sont disponibles, une page de sélection s'affiche à l'ouverture du formulaire, listant tous les dashboards par nom.

Les modules Python ne sont jamais publiés sur GitHub — ils restent sur le serveur et se transfèrent uniquement via le bouton de téléchargement/chargement dans les Paramètres.
