{# 1. Overview #}

1. Platform overview

KoboDashBoard is a web application that connects to a KoboToolBox server and displays form submission data as interactive dashboards.

The platform is organised into groups: each group has a set of accessible forms and a member list. A user only sees the forms belonging to their groups.

RoleAccess
User View dashboards for their groups, use filters and exports.
Group admin Everything a user can do + manage their group's members + upload dashboard modules.
Main administrator Full access: server configuration, all forms, groups and users.
{# 2. Access and login #}

2. Access and login

Requesting access

If you don't have an account yet, click "Create an account" on the login page. Enter your full name and email address, then choose a password. Your request will be sent to an administrator for approval. You will receive a confirmation message once your account is activated.

Logging in

Enter your email address and password on the login page (/accounts/login/). The session stays open until you click "Log out" in the top right.

Forgotten password

Contact your group administrator or the main administrator. They can generate a password reset link and send it to you directly.

{# 3. Dashboards #}

3. Viewing dashboards

The home page ("Forms") lists all dashboards you have access to. Click "View dashboard" to open a form.

Three types of views exist depending on the form:

  • Configured dashboard — built via the dashboard editor (see section 8). Displays widgets defined by the administrator: summary numbers, pie charts, bar/line charts, data tables, with custom filters.
  • Indicator dashboard — available if a specific Python module has been loaded for the form. Displays summary cards, charts by country/period/result, disaggregation and a detailed data table.
  • Coverage matrix — available if a specific Python module has been loaded. Displays a cross-table of activities × countries with the number of submissions per combination. Clickable cells give access to the corresponding submission list.

The Refresh button (on the dashboard page) forces a reload of data from the KoboToolBox server, bypassing the cache.

{# 4. Filters #}

4. Using filters

Dashboards offer a filter bar at the top of the page. Available filters vary by form type:

  • Country — restrict display to a country or geographic area.
  • Year / Quarter — filter by reporting period.
  • Result — show only the indicators for a given result.
  • Responsible — filter by the person or entity responsible for the activity (coverage matrix).

Filters are cumulative. To reset them, click "Reset" or remove the parameters from the URL.

{# 5. Exports #}

5. Exporting data

Two export formats are available from the dashboard page:

  • CSV — comma-separated text file, compatible with most spreadsheet applications. UTF-8 with BOM encoding for optimal Excel compatibility.
  • XLSX — native Excel file with formatted headers and auto-adjusted columns.

The export covers all form data, not just the rows filtered on screen.

{# 6. Group admins #}

6. Group administrators

If you are a group administrator, the icon appears in the navigation bar. It gives access to the "My group" page.

Managing members
  • Add a member — select an active user from the dropdown and click "Add". Only users already created and activated by the main administrator appear in the list.
  • Deactivate an account — the user can no longer log in but their account is kept. Click .
  • Delete an account — permanent deletion. Click and confirm.
  • Reset password — click to generate a one-time reset link (valid 3 days). Copy this link and send it to the user by email or messaging.
Dashboard editor

As a group administrator, you can create and edit visual dashboards for your group's forms via the dashboard editor. Access it from the Settings page or the form list. See section 8 for details.

Managing dashboard modules

Each form can have a Python module file (.py) that defines how submissions are analysed and displayed. As a group admin, you can:

  • Download the current module — click Module to retrieve the .py file currently in use.
  • Upload a new module — select a .py file and click Upload. The new module takes effect immediately (no server restart in development; a gunicorn restart is required in production).
The module file must be a valid Python script with a valid Python identifier as its filename (letters, digits, underscores — no spaces or hyphens).
{# 7. Main administrator #}

7. Main administrator

The main administrator accesses the Settings page ( in the navigation bar).

KoboToolBox server connection

Enter the server URL (e.g. https://kobo.ifrc.org) and the API token. The token is available on your KoboToolBox profile at <server-url>/token/?format=json. Click "Test & load forms" to verify the connection and get the list of available forms.

Adding a form

After loading the form list, select one from the dropdown and click "Add this form". You can adjust the cache duration (in seconds) to control how often data is synchronised with KoboToolBox.

Managing groups

Create named groups to organise access to forms. For each group, you can:

  • Assign forms (group members will only see those forms).
  • Assign members (active users).
  • Designate group administrators from among the members.

Forms not assigned to any group are visible only to the main administrator.

Managing users

The "User management" page () lists accounts awaiting approval and active accounts. You can:

  • Activate a new account after verifying identity.
  • Deactivate an account temporarily without deleting it.
  • Delete an account permanently.
  • Generate a password reset link () — send it to the user via the channel of your choice. The link is valid for 3 days and is single-use.
Identity and colour

From Settings, in the "Identity" card, you can:

  • Organisation name — displayed in the centre of the navigation bar instead of "KoboDashBoard".
  • Logo — replaces the text in the centre of the navigation bar. Recommended formats: transparent PNG or SVG, 48 px height.
  • Main colour — use the colour picker or enter a hex code (e.g. #00833D). The "Default" button restores Bootstrap red. The change applies instantly to all users.
Interface language

A dropdown in the navigation bar lets you change the interface language. Five languages are available: Français, English, Español, العربية (with automatic right-to-left layout), Русский. The choice is remembered for the current session. The same selector is available on the login and registration pages.

Access recovery

In case of a lost password, connect to the server via SSH and run: 

python3 manage.py init_admin

A one-time login link (valid 3 days) is displayed in the terminal.

{# 8. Dashboard editor #}

8. Dashboard editor

The dashboard editor lets administrators create visual dashboards for any form without writing code. Multiple dashboards can coexist for the same form.

The editor is accessible from the Settings page by clicking "Dashboard editor" next to the relevant form. It is available to the main administrator and group administrators whose group includes that form.

Dashboard list

The list shows all dashboards created for the form. You can create a new one by giving it a name. For each existing dashboard, two actions are available: View (read-only display) and Edit (open in the editor).

Dashboard structure

A dashboard is made up of rows stacked vertically. Each row can contain 1, 2 or 3 columns. Rows can be reordered (▲▼ arrows) or deleted.

At the top of the editor, you can:

  • Rename the dashboard.
  • Configure filters — select form fields that will appear as filters at the top of the dashboard (e.g. country, period). Filters can be reordered and deleted. A custom label can be entered for each one.
Adding and editing widgets

Each row can contain widgets. Click "Add a widget" in a row to open the add form. Four widget types are available:

TypeDescriptionMain options
Summary number Displays a single value in large characters. Submission count, sum or average of a numeric field.
Pie chart Pie chart grouped by values of a field. Customisable colour and label per value.
Bar / line chart Bars or lines with one or more series fields. X axis (optional), style (bars / lines), stacked mode, ordered series with colour and label.
Data table Displays submissions as a table. Select which columns to display.

To edit an existing widget, click the icon on its card. The edit form appears inline. To delete a widget, click and confirm.

Configuring series (bar / line chart)

The bar / line chart widget can display multiple fields as series on the same chart. In the Series section:

  • Select a field from the dropdown and click "+ Add".
  • Each series appears in the list with a colour picker and a label field (displayed in the legend).
  • Reorder series with the ▲▼ arrows.
  • Delete a series with the × button.

For fixed-choice fields (select_one), an additional "Colours by value" section lets you assign a specific colour to each category (e.g. Male / Female / Other).

Changes in the editor are saved immediately with each action (add, delete, reorder). The Preview button at the top of the editor lets you see the dashboard as users will see it.
{# 9. Python modules #}

9. Advanced dashboards with Python modules

The dashboard editor covers common use cases. For more complex analyses — specific business logic, custom aggregations, coverage matrices — it is possible to create a Python module with the help of Claude Code.

How a module works

A module is a .py file placed in the form_modules/ directory on the server. It is linked to a form via its KoboToolBox UID and defines how submissions are analysed and displayed. Two styles are possible:

  • Indicator dashboard — the module implements parse_submissions(). Displays summary cards and charts by country, period and result.
  • Coverage matrix — the module implements parse_structure(). Displays a cross-table of activities × countries with the number of submissions per combination.
Creating a module with Claude Code
  1. Open Claude Code in the KoboDashBoard project directory.
  2. Describe the form, its KoboToolBox UID and the desired logic.
  3. Claude Code creates form_modules/<name>.py based on the FormModule class.
  4. Restart the development server — the module registers automatically.
  5. Test the dashboard locally before deploying.
Exporting and importing a module

Module files are not included in the Git repository (to avoid exposing organisation-specific data). To deploy a module to production:

  1. Download from the development server — in Settings, click Download module next to the form.
  2. Import to production — on the production server, in Settings, click Upload module and select the downloaded file.
  3. Restart gunicorn: sudo systemctl restart kobodashboard (or use the deploy script).
Multiple dashboards for the same form

A form can have both dashboards created via the editor and a Python module. When multiple options are available, a selection page is displayed when opening the form, listing all dashboards by name.

Python modules are never published to GitHub — they remain on the server and are transferred only via the download/upload button in Settings.
