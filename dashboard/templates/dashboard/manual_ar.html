{# 1. نظرة عامة #}

١. نظرة عامة على المنصة

KoboDashBoard هو تطبيق ويب يتصل بخادم KoboToolBox ويعرض بيانات تقديم النماذج في شكل لوحات تحكم تفاعلية.

تُنظَّم المنصة في مجموعات: تمتلك كل مجموعة مجموعة من النماذج المتاحة وقائمة بالأعضاء. لا يرى المستخدم إلا النماذج التابعة لمجموعاته.

الدورالصلاحيات
مستخدم عرض لوحات تحكم مجموعاته، استخدام المرشحات والتصدير.
مدير مجموعة كل ما يستطيع المستخدم فعله + إدارة أعضاء مجموعته + تحميل وحدات لوحة التحكم.
المدير الرئيسي وصول كامل: إعداد الخادم، إدارة جميع النماذج والمجموعات والمستخدمين.
{# 2. الوصول #}

٢. الوصول وتسجيل الدخول

طلب الوصول

إذا لم يكن لديك حساب بعد، انقر على «إنشاء حساب» في صفحة تسجيل الدخول. أدخل اسمك الكامل وعنوان بريدك الإلكتروني، ثم اختر كلمة مرور. سيُرسَل طلبك إلى مدير للموافقة عليه. ستتلقى رسالة تأكيد عند تفعيل حسابك.

تسجيل الدخول

أدخل عنوان بريدك الإلكتروني وكلمة المرور في صفحة تسجيل الدخول (/accounts/login/). تبقى الجلسة مفتوحة حتى تنقر على «تسجيل الخروج» في أعلى اليمين.

نسيان كلمة المرور

تواصل مع مدير مجموعتك أو المدير الرئيسي. بإمكانه إنشاء رابط إعادة تعيين كلمة المرور وإرساله إليك مباشرةً.

{# 3. لوحات التحكم #}

٣. عرض لوحات التحكم

تعرض الصفحة الرئيسية («النماذج») جميع لوحات التحكم التي يمكنك الوصول إليها. انقر على «عرض لوحة التحكم» لفتح نموذج.

توجد ثلاثة أنواع من العروض بحسب النموذج:

  • لوحة تحكم مُهيَّأة — مبنية عبر محرر لوحة التحكم (انظر القسم ٨). تعرض الأدوات التي حددها المدير: أرقاماً ملخصة، مخططات دائرية أو أعمدة، جداول بيانات، مع مرشحات مخصصة.
  • لوحة تحكم المؤشرات — متاحة إذا تم تحميل وحدة Python خاصة للنموذج. تعرض بطاقات ملخص ومخططات حسب البلد/الفترة/النتيجة، وتفصيلاً وجدول بيانات مفصلاً.
  • مصفوفة التغطية — متاحة إذا تم تحميل وحدة Python خاصة. تعرض جدولاً متقاطعاً للأنشطة × البلدان مع عدد التقديمات لكل تركيبة. الخلايا قابلة للنقر وتتيح الوصول إلى قائمة التقديمات المقابلة.

يُجبر زر تحديث (في صفحة لوحة التحكم) على إعادة تحميل البيانات من خادم KoboToolBox، متجاهلاً الذاكرة المؤقتة.

{# 4. المرشحات #}

٤. استخدام المرشحات

تقدم لوحات التحكم شريط مرشحات في أعلى الصفحة. تتفاوت المرشحات المتاحة بحسب نوع النموذج:

  • البلد — تقييد العرض ببلد أو منطقة جغرافية.
  • السنة / الربع — التصفية حسب فترة التقرير.
  • النتيجة — عرض مؤشرات نتيجة معينة فقط.
  • المسؤول — التصفية حسب الشخص أو الجهة المسؤولة عن النشاط (مصفوفة التغطية).

المرشحات تراكمية. لإعادة تعيينها، انقر على «إعادة التعيين» أو احذف المعاملات من عنوان URL.

{# 5. التصدير #}

٥. تصدير البيانات

يتوفر صيغتان للتصدير من صفحة لوحة التحكم:

  • CSV — ملف نصي مفصول بفواصل، متوافق مع معظم برامج جداول البيانات. ترميز UTF-8 مع BOM لتوافق مثالي مع Excel.
  • XLSX — ملف Excel أصلي بترويسات منسقة وأعمدة ضبط تلقائي.

يشمل التصدير جميع بيانات النموذج، وليس فقط الصفوف المرشحة على الشاشة.

{# 6. مديرو المجموعات #}

٦. مديرو المجموعات

إذا كنت مدير مجموعة، يظهر أيقونة في شريط التنقل. يتيح الوصول إلى صفحة «مجموعتي».

إدارة الأعضاء
  • إضافة عضو — حدد مستخدماً نشطاً من القائمة المنسدلة وانقر على «إضافة». تظهر فقط المستخدمون الذين أنشأهم وفعّلهم المدير الرئيسي.
  • تعطيل حساب — لا يستطيع المستخدم تسجيل الدخول بعد الآن لكن حسابه يُحفظ. انقر على .
  • حذف حساب — حذف نهائي. انقر على وأكّد.
  • إعادة تعيين كلمة المرور — انقر على لإنشاء رابط إعادة تعيين يُستخدم مرة واحدة (صالح ٣ أيام). انسخ الرابط وأرسله إلى المستخدم عبر البريد الإلكتروني أو المراسلة.
محرر لوحة التحكم

بوصفك مدير مجموعة، يمكنك إنشاء وتعديل لوحات تحكم مرئية لنماذج مجموعتك عبر محرر لوحة التحكم. يمكن الوصول إليه من صفحة الإعدادات أو قائمة النماذج. انظر القسم ٨ للتفاصيل.

إدارة وحدات لوحة التحكم

يمكن أن يحتوي كل نموذج على ملف وحدة Python (.py) يحدد كيفية تحليل التقديمات وعرضها. بوصفك مدير مجموعة، يمكنك:

  • تنزيل الوحدة الحالية — انقر على الوحدة لاسترداد ملف .py قيد الاستخدام.
  • تحميل وحدة جديدة — حدد ملف .py وانقر على تحميل. تسري الوحدة الجديدة فوراً (دون إعادة تشغيل الخادم في وضع التطوير؛ يلزم إعادة تشغيل gunicorn في الإنتاج).
يجب أن يكون ملف الوحدة سكريبت Python صالحاً باسم ملف يمثل معرفاً Python صالحاً (أحرف وأرقام وشرطات سفلية — بدون مسافات أو شرطات).
{# 7. المدير الرئيسي #}

٧. المدير الرئيسي

يصل المدير الرئيسي إلى صفحة الإعدادات ( في شريط التنقل).

الاتصال بخادم KoboToolBox

أدخل URL الخادم (مثل https://kobo.ifrc.org) ورمز API. يتوفر الرمز في ملف تعريف KoboToolBox الخاص بك على <url-الخادم>/token/?format=json. انقر على «اختبار وتحميل النماذج» للتحقق من الاتصال والحصول على قائمة النماذج المتاحة.

إضافة نموذج

بعد تحميل قائمة النماذج، حدد نموذجاً من القائمة المنسدلة وانقر على «إضافة هذا النموذج». يمكنك ضبط مدة الذاكرة المؤقتة (بالثواني) للتحكم في تكرار المزامنة مع KoboToolBox.

إدارة المجموعات

أنشئ مجموعات مسماة لتنظيم الوصول إلى النماذج. لكل مجموعة، يمكنك:

  • تعيين النماذج (سيرى أعضاء المجموعة تلك النماذج فقط).
  • تعيين الأعضاء (المستخدمون النشطون).
  • تعيين مديري مجموعات من بين الأعضاء.

النماذج غير المعينة لأي مجموعة تكون مرئية فقط للمدير الرئيسي.

إدارة المستخدمين

تعرض صفحة «إدارة المستخدمين» () الحسابات المعلقة للموافقة والحسابات النشطة. يمكنك:

  • تفعيل حساب جديد بعد التحقق من الهوية.
  • تعطيل حساب مؤقتاً دون حذفه.
  • حذف حساب نهائياً.
  • إنشاء رابط إعادة تعيين كلمة المرور () — أرسله إلى المستخدم عبر القناة التي تختارها. الرابط صالح ٣ أيام ويُستخدم مرة واحدة.
الهوية واللون

من الإعدادات، في بطاقة «الهوية»، يمكنك:

  • اسم المنظمة — يظهر في وسط شريط التنقل بدلاً من «KoboDashBoard».
  • الشعار — يحل محل النص في وسط شريط التنقل. الصيغ الموصى بها: PNG أو SVG شفاف، ارتفاع ٤٨ بكسل.
  • اللون الرئيسي — استخدم منتقي الألوان أو أدخل رمزاً سداسياً (مثل #00833D). يستعيد زر «الافتراضي» الأحمر Bootstrap. يسري التغيير فوراً على جميع المستخدمين.
لغة الواجهة

تتيح قائمة منسدلة في شريط التنقل تغيير لغة الواجهة. خمس لغات متاحة: Français، English، Español، العربية (مع تخطيط من اليمين إلى اليسار تلقائياً)، Русский. يُحفظ الاختيار للجلسة الحالية. يتوفر نفس المحدد في صفحتي تسجيل الدخول والتسجيل.

استرداد الوصول

في حال فقدان كلمة المرور، اتصل بالخادم عبر SSH ونفذ: 

python3 manage.py init_admin

يظهر رابط تسجيل دخول يُستخدم مرة واحدة (صالح ٣ أيام) في الطرفية.

{# 8. محرر لوحة التحكم #}

٨. محرر لوحة التحكم

يُتيح محرر لوحة التحكم للمديرين إنشاء لوحات تحكم مرئية لأي نموذج دون كتابة أي كود. يمكن أن تتعايش عدة لوحات تحكم لنفس النموذج.

يمكن الوصول إلى المحرر من صفحة الإعدادات بالنقر على «محرر لوحة التحكم» بجانب النموذج المعني. يتوفر للمدير الرئيسي ومديري المجموعات الذين تشمل مجموعاتهم ذلك النموذج.

قائمة لوحات التحكم

تعرض القائمة جميع لوحات التحكم المنشأة للنموذج. يمكنك إنشاء واحدة جديدة بإعطائها اسماً. لكل لوحة تحكم موجودة، يتوفر إجراءان: عرض (عرض للقراءة فقط) وتعديل (فتح في المحرر).

بنية لوحة التحكم

تتألف لوحة التحكم من صفوف مكدسة عمودياً. يمكن أن يحتوي كل صف على ١ أو ٢ أو ٣ أعمدة. يمكن إعادة ترتيب الصفوف (أسهم ▲▼) أو حذفها.

في أعلى المحرر، يمكنك:

  • إعادة تسمية لوحة التحكم.
  • تهيئة المرشحات — حدد حقول النموذج التي ستظهر كمرشحات في أعلى لوحة التحكم (مثل البلد، الفترة). يمكن إعادة ترتيب المرشحات وحذفها. يمكن إدخال تسمية مخصصة لكل منها.
إضافة الأدوات وتعديلها

يمكن أن يحتوي كل صف على أدوات. انقر على «إضافة أداة» في صف لفتح نموذج الإضافة. يتوفر أربعة أنواع من الأدوات:

النوعالوصفالخيارات الرئيسية
رقم ملخص يعرض قيمة واحدة بأحرف كبيرة. عدد التقديمات، مجموع أو متوسط حقل رقمي.
مخطط دائري مخطط دائري مجمع حسب قيم حقل. لون وتسمية قابلان للتخصيص لكل قيمة.
مخطط أعمدة / خطوط أعمدة أو خطوط بحقل سلسلة واحدة أو أكثر. محور X (اختياري)، النمط (أعمدة / خطوط)، الوضع المكدس، سلاسل مرتبة مع اللون والتسمية.
جدول بيانات يعرض التقديمات في شكل جدول. اختيار الأعمدة المراد عرضها.

لتعديل أداة موجودة، انقر على أيقونة في بطاقتها. يظهر نموذج التعديل مضمناً. لحذف أداة، انقر على وأكّد.

تهيئة السلاسل (مخطط الأعمدة / الخطوط)

تتيح أداة «مخطط الأعمدة / الخطوط» عرض عدة حقول كسلاسل على نفس المخطط. في قسم السلاسل:

  • حدد حقلاً من القائمة المنسدلة وانقر على «+ إضافة».
  • تظهر كل سلسلة في القائمة مع منتقي لون وحقل تسمية (يظهر في وسيلة الإيضاح).
  • أعد ترتيب السلاسل بالأسهم ▲▼.
  • احذف سلسلة بزر ×.

بالنسبة لحقول الاختيار الثابت (select_one)، يتيح قسم إضافي «ألوان حسب القيمة» تعيين لون محدد لكل فئة (مثل ذكر / أنثى / آخر).

تُحفظ التغييرات في المحرر فوراً مع كل إجراء (إضافة، حذف، إعادة ترتيب). يُتيح زر معاينة في أعلى المحرر رؤية لوحة التحكم كما ستظهر للمستخدمين.
{# 9. وحدات Python #}

٩. لوحات تحكم متقدمة بوحدات Python

يغطي محرر لوحة التحكم الحالات الشائعة. للتحليلات الأكثر تعقيداً — منطق عمل خاص، تجميعات مخصصة، مصفوفات تغطية — يمكن إنشاء وحدة Python بمساعدة Claude Code.

كيف تعمل الوحدة

الوحدة هي ملف .py موجود في دليل form_modules/ على الخادم. ترتبط بنموذج عبر UID KoboToolBox الخاص به وتحدد كيفية تحليل التقديمات وعرضها. نمطان ممكنان:

  • لوحة تحكم المؤشرات — تُطبّق الوحدة parse_submissions(). تعرض بطاقات ملخص ومخططات حسب البلد والفترة والنتيجة.
  • مصفوفة التغطية — تُطبّق الوحدة parse_structure(). تعرض جدولاً متقاطعاً للأنشطة × البلدان مع عدد التقديمات لكل تركيبة.
إنشاء وحدة مع Claude Code
  1. افتح Claude Code في دليل مشروع KoboDashBoard.
  2. صف النموذج، وUID KoboToolBox الخاص به، والمنطق المطلوب.
  3. ينشئ Claude Code الملف form_modules/<الاسم>.py بناءً على فئة FormModule.
  4. أعد تشغيل خادم التطوير — تتسجل الوحدة تلقائياً.
  5. اختبر لوحة التحكم محلياً قبل نشرها.
تصدير الوحدة واستيرادها

لا تُدرَج ملفات الوحدات في مستودع Git (لتجنب الكشف عن بيانات خاصة بالمنظمة). لنشر وحدة في الإنتاج:

  1. التنزيل من خادم التطوير — في الإعدادات، انقر على تنزيل الوحدة بجانب النموذج.
  2. الاستيراد إلى الإنتاج — على خادم الإنتاج، في الإعدادات، انقر على تحميل الوحدة وحدد الملف المنزَّل.
  3. أعد تشغيل gunicorn: sudo systemctl restart kobodashboard (أو استخدم سكريبت النشر).
لوحات تحكم متعددة لنفس النموذج

يمكن أن يمتلك النموذج لوحات تحكم منشأة عبر المحرر ووحدة Python في آنٍ واحد. عند توفر خيارات متعددة، تظهر صفحة اختيار عند فتح النموذج، تسرد جميع لوحات التحكم بأسمائها.

لا تُنشر وحدات Python أبداً على GitHub — تبقى على الخادم وتُنقل فقط عبر زر التنزيل/التحميل في الإعدادات.
