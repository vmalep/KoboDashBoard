{# 1. Обзор #}

1. Обзор платформы

KoboDashBoard — веб-приложение, которое подключается к серверу KoboToolBox и отображает данные отправок форм в виде интерактивных панелей управления.

Платформа организована по группам: у каждой группы есть набор доступных форм и список участников. Пользователь видит только формы, принадлежащие его группам.

РольДоступ
Пользователь Просмотр панелей управления своих групп, использование фильтров и экспорта.
Администратор группы Всё, что может пользователь + управление участниками группы + загрузка модулей панелей.
Главный администратор Полный доступ: настройка сервера, управление всеми формами, группами и пользователями.
{# 2. Доступ #}

2. Доступ и вход в систему

Запрос доступа

Если у вас ещё нет учётной записи, нажмите «Создать учётную запись» на странице входа. Введите полное имя и адрес электронной почты, затем выберите пароль. Ваш запрос будет отправлен администратору для одобрения. После активации учётной записи вы получите сообщение с подтверждением.

Вход в систему

Введите адрес электронной почты и пароль на странице входа (/accounts/login/). Сессия остаётся открытой, пока вы не нажмёте «Выйти» в правом верхнем углу.

Забытый пароль

Обратитесь к администратору вашей группы или главному администратору. Он может сгенерировать ссылку для сброса пароля и отправить её вам напрямую.

{# 3. Панели управления #}

3. Просмотр панелей управления

Главная страница («Формы») перечисляет все панели управления, к которым у вас есть доступ. Нажмите «Просмотр панели», чтобы открыть форму.

Существует три типа отображения в зависимости от формы:

  • Настроенная панель управления — создана через редактор панелей (см. раздел 8). Отображает виджеты, заданные администратором: сводные числа, круговые диаграммы, столбчатые диаграммы, таблицы данных с пользовательскими фильтрами.
  • Панель индикаторов — доступна, если для формы загружен специальный Python-модуль. Отображает сводные карточки, диаграммы по странам/периодам/результатам, дезагрегацию и подробную таблицу данных.
  • Матрица охвата — доступна, если загружен специальный Python-модуль. Отображает перекрёстную таблицу мероприятий × стран с количеством отправок на каждую комбинацию. Кликабельные ячейки открывают список соответствующих отправок.

Кнопка Обновить (на странице панели) принудительно перезагружает данные с сервера KoboToolBox, игнорируя кэш.

{# 4. Фильтры #}

4. Использование фильтров

Панели управления предлагают панель фильтров в верхней части страницы. Доступные фильтры варьируются в зависимости от типа формы:

  • Страна — ограничить отображение страной или географической зоной.
  • Год / Квартал — фильтрация по периоду отчётности.
  • Результат — показывать только индикаторы заданного результата.
  • Ответственный — фильтрация по лицу или структуре, ответственным за мероприятие (матрица охвата).

Фильтры можно применять одновременно. Для сброса нажмите «Сбросить» или удалите параметры из URL.

{# 5. Экспорт #}

5. Экспорт данных

На странице панели управления доступны два формата экспорта:

  • CSV — текстовый файл с разделителями-запятыми, совместимый с большинством табличных процессоров. Кодировка UTF-8 с BOM для оптимальной совместимости с Excel.
  • XLSX — собственный формат Excel с форматированными заголовками и автоматически настроенными столбцами.

Экспорт охватывает все данные формы, а не только строки, отфильтрованные на экране.

{# 6. Администраторы групп #}

6. Администраторы групп

Если вы являетесь администратором группы, на панели навигации появляется значок . Он открывает страницу «Моя группа».

Управление участниками
  • Добавить участника — выберите активного пользователя из выпадающего списка и нажмите «Добавить». В списке отображаются только пользователи, созданные и активированные главным администратором.
  • Деактивировать учётную запись — пользователь больше не сможет войти в систему, но его учётная запись сохраняется. Нажмите .
  • Удалить учётную запись — окончательное удаление. Нажмите и подтвердите.
  • Сброс пароля — нажмите , чтобы сгенерировать одноразовую ссылку для сброса (действительна 3 дня). Скопируйте ссылку и отправьте её пользователю по электронной почте или через мессенджер.
Редактор панели управления

Как администратор группы, вы можете создавать и редактировать визуальные панели управления для форм вашей группы через редактор панелей. Откройте его со страницы Настройки или из списка форм. Подробности в разделе 8.

Управление модулями панелей

Каждая форма может иметь Python-файл модуля (.py), определяющий способ анализа и отображения отправок. Как администратор группы, вы можете:

  • Скачать текущий модуль — нажмите Модуль, чтобы получить используемый файл .py.
  • Загрузить новый модуль — выберите файл .py и нажмите Загрузить. Новый модуль вступает в силу немедленно (без перезапуска сервера в режиме разработки; в production требуется перезапуск gunicorn).
Файл модуля должен быть корректным Python-скриптом с допустимым Python-идентификатором в качестве имени файла (буквы, цифры, подчёркивания — без пробелов и дефисов).
{# 7. Главный администратор #}

7. Главный администратор

Главный администратор открывает страницу Настройки ( на панели навигации).

Подключение к серверу KoboToolBox

Укажите URL сервера (например, https://kobo.ifrc.org) и токен API. Токен доступен в вашем профиле KoboToolBox по адресу <url-сервера>/token/?format=json. Нажмите «Проверить и загрузить формы», чтобы проверить подключение и получить список доступных форм.

Добавление формы

После загрузки списка форм выберите нужную из выпадающего меню и нажмите «Добавить эту форму». Вы можете настроить продолжительность кэша (в секундах), чтобы контролировать частоту синхронизации с KoboToolBox.

Управление группами

Создайте именованные группы для организации доступа к формам. Для каждой группы вы можете:

  • Назначать формы (участники группы будут видеть только эти формы).
  • Назначать участников (активных пользователей).
  • Назначать администраторов группы из числа её участников.

Формы, не назначенные ни одной группе, видны только главному администратору.

Управление пользователями

Страница «Управление пользователями» () отображает учётные записи, ожидающие одобрения, и активные учётные записи. Вы можете:

  • Активировать новую учётную запись после проверки личности.
  • Деактивировать учётную запись временно, не удаляя её.
  • Удалить учётную запись окончательно.
  • Сгенерировать ссылку для сброса пароля () — отправьте её пользователю удобным способом. Ссылка действительна 3 дня и является одноразовой.
Идентификация и цвет

В Настройках, в карточке «Идентификация», вы можете:

  • Название организации — отображается в центре панели навигации вместо «KoboDashBoard».
  • Логотип — заменяет текст в центре панели навигации. Рекомендуемые форматы: прозрачный PNG или SVG, высота 48 пикселей.
  • Основной цвет — используйте выбор цвета или введите шестнадцатеричный код (например, #00833D). Кнопка «По умолчанию» восстанавливает красный Bootstrap. Изменение применяется мгновенно для всех пользователей.
Язык интерфейса

Выпадающее меню на панели навигации позволяет изменить язык интерфейса. Доступно пять языков: Français, English, Español, العربية (с автоматическим макетом справа налево), Русский. Выбор сохраняется для текущей сессии. Тот же переключатель доступен на страницах входа и регистрации.

Восстановление доступа

В случае утери пароля подключитесь к серверу по SSH и выполните: 

python3 manage.py init_admin

В терминале отобразится одноразовая ссылка для входа (действительна 3 дня).

{# 8. Редактор панели #}

8. Редактор панели управления

Редактор панелей управления позволяет администраторам создавать визуальные панели для любой формы без написания кода. Для одной формы может сосуществовать несколько панелей управления.

Редактор доступен со страницы Настройки — нажмите «Редактор панели» рядом с нужной формой. Он доступен главному администратору и администраторам групп, чья группа включает эту форму.

Список панелей управления

В списке отображаются все панели управления, созданные для формы. Вы можете создать новую, задав ей название. Для каждой существующей панели доступны два действия: Просмотр (только для чтения) и Редактировать (открыть в редакторе).

Структура панели управления

Панель управления состоит из строк, расположенных вертикально. Каждая строка может содержать 1, 2 или 3 столбца. Строки можно переупорядочивать (стрелки ▲▼) или удалять.

В верхней части редактора вы можете:

  • Переименовать панель управления.
  • Настроить фильтры — выберите поля формы, которые будут отображаться как фильтры в верхней части панели (например, страна, период). Фильтры можно переупорядочивать и удалять. Для каждого можно ввести пользовательскую метку.
Добавление и редактирование виджетов

Каждая строка может содержать виджеты. Нажмите «Добавить виджет» в строке, чтобы открыть форму добавления. Доступно четыре типа виджетов:

ТипОписаниеОсновные параметры
Сводное число Отображает одно значение крупным шрифтом. Количество отправок, сумма или среднее числового поля.
Круговая диаграмма Круговая диаграмма, сгруппированная по значениям поля. Настраиваемый цвет и метка для каждого значения.
Столбчатая / линейная диаграмма Столбцы или линии с одной или несколькими рядами полей. Ось X (необязательно), стиль (столбцы / линии), режим наложения, упорядоченные ряды с цветом и меткой.
Таблица данных Отображает отправки в виде таблицы. Выбор отображаемых столбцов.

Чтобы отредактировать существующий виджет, нажмите значок на его карточке. Форма редактирования появится встроенной. Чтобы удалить виджет, нажмите и подтвердите.

Настройка рядов (столбчатая / линейная диаграмма)

Виджет «Столбчатая / линейная диаграмма» позволяет отображать несколько полей как ряды на одной диаграмме. В разделе Ряды:

  • Выберите поле из выпадающего списка и нажмите «+ Добавить».
  • Каждый ряд появляется в списке с выбором цвета и полем метка (отображается в легенде).
  • Переупорядочивайте ряды стрелками ▲▼.
  • Удаляйте ряд кнопкой ×.

Для полей с фиксированным выбором (select_one) дополнительный раздел «Цвета по значению» позволяет назначить конкретный цвет каждой категории (например, Мужчина / Женщина / Другое).

Изменения в редакторе сохраняются немедленно при каждом действии (добавление, удаление, переупорядочивание). Кнопка Предпросмотр в верхней части редактора позволяет увидеть панель такой, какой её увидят пользователи.
{# 9. Python-модули #}

9. Расширенные панели с Python-модулями

Редактор панелей охватывает стандартные случаи. Для более сложной аналитики — специфической бизнес-логики, настраиваемых агрегаций, матриц охвата — можно создать Python-модуль с помощью Claude Code.

Как работает модуль

Модуль — это файл .py, расположенный в директории form_modules/ на сервере. Он связан с формой через её UID в KoboToolBox и определяет способ анализа и отображения отправок. Возможны два стиля:

  • Панель индикаторов — модуль реализует parse_submissions(). Отображает сводные карточки и диаграммы по странам, периодам и результатам.
  • Матрица охвата — модуль реализует parse_structure(). Отображает перекрёстную таблицу мероприятий × стран с количеством отправок на каждую комбинацию.
Создание модуля с Claude Code
  1. Откройте Claude Code в директории проекта KoboDashBoard.
  2. Опишите форму, её UID в KoboToolBox и желаемую логику.
  3. Claude Code создаёт файл form_modules/<имя>.py на основе класса FormModule.
  4. Перезапустите сервер разработки — модуль зарегистрируется автоматически.
  5. Протестируйте панель локально перед развёртыванием.
Экспорт и импорт модуля

Файлы модулей не включаются в репозиторий Git (чтобы не раскрывать данные организации). Для развёртывания модуля в production:

  1. Скачать с сервера разработки — в Настройках нажмите Скачать модуль рядом с формой.
  2. Импортировать в production — на production-сервере в Настройках нажмите Загрузить модуль и выберите скачанный файл.
  3. Перезапустите gunicorn: sudo systemctl restart kobodashboard (или используйте скрипт развёртывания).
Несколько панелей для одной формы

Форма может иметь как панели, созданные через редактор, так и Python-модуль. При наличии нескольких вариантов при открытии формы отображается страница выбора, перечисляющая все панели по названиям.

Python-модули никогда не публикуются на GitHub — они остаются на сервере и передаются только через кнопку скачивания/загрузки в Настройках.
